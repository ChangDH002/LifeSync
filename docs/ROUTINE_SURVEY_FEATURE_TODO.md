# Routine & Survey Feature TODO

현재 코드 기준으로 설문, 맞춤 루틴, 마이페이지 연동은 일부만 연결된 상태다.
이 문서는 기능 관점에서 프론트엔드와 백엔드의 남은 작업을 나눈 TODO 목록이다.

## 현재 상태 요약

- 치매 위험도 설문 페이지는 존재한다.
- 설문 결과는 백엔드에 저장되도록 연결되어 있다.
- 마이페이지에는 최신 설문 결과가 일부 반영된다.
- 루틴 페이지와 루틴 조회 API는 존재한다.
- 그러나 맞춤 루틴은 아직 사용자별 추천이 아니라 정적 루틴 3개 기반이다.

## Frontend TODO

### 1. 설문 UX

- 설문 시작 버튼 진입 경로를 전체 화면에서 점검하고 누락된 CTA를 모두 연결
- 설문 진행 중 이전 질문으로 이동하는 UX 정리
- 설문 중도 이탈 시 임시 저장 여부 결정
- 설문 완료 후 저장 성공, 실패, 로그인 필요 상태를 더 명확하게 표시
- 설문 결과 화면에 추천 이유와 다음 행동을 더 구체적으로 노출

### 2. 설문 결과 활용

- 설문 결과 화면에서 위험도와 영역별 점수 설명을 더 구조화
- 위험도별 추천 행동 문구를 기획안에 맞게 정리
- 설문 완료 후 루틴 페이지로 이동할 때 실제 추천 루틴 컨텍스트를 유지
- 재설문 시 이전 결과와 비교 표시 여부 결정

### 3. 맞춤 루틴 화면

- 오늘의 루틴 목록에 완료 버튼 추가
- 루틴 완료/취소 시 즉시 UI 반영
- 루틴별 추천 이유 표시
- 루틴 카테고리, 빈도, 우선순위 표시
- 오늘 루틴, 주간 루틴, 완료 기록 탭 분리 검토
- 추천 루틴이 정적 예시인지 실제 추천인지 더 명확히 표시

### 4. 마이페이지 연동

- 설문 결과 요약과 루틴 추천 결과를 더 자연스럽게 연결
- 오늘 루틴 완료 수와 주간 달성률을 실제 완료 데이터와 연동
- 최근 활동에 설문 저장, 루틴 완료, 루틴 갱신 이벤트를 함께 표시
- 설문 결과에 따라 배너 문구와 CTA를 다르게 노출

### 5. 예외 처리

- 비로그인 사용자가 설문을 완료했을 때 저장 유도 UX 정리
- 설문 저장 실패 시 재시도 버튼 제공
- 루틴 API 실패 시 fallback 표시를 더 사용자 친화적으로 개선
- 빈 데이터 상태와 최초 사용자 상태를 구분

### 6. QA

- 설문 시작부터 완료까지 실제 브라우저 플로우 점검
- 설문 저장 후 마이페이지 반영 확인
- 설문 결과 후 루틴 페이지 이동 플로우 점검
- 모바일 화면에서 설문/루틴 UI 점검

## Backend TODO

### 1. 설문 저장 API 고도화

- 프론트가 보낸 점수를 그대로 저장하지 말고 서버에서 재계산/검증
- 설문 질문 세트 버전 관리 추가
- 중복 제출 정책 정의
- 최신 결과만 유지할지 이력 전체를 저장할지 정책 확정
- 설문 결과 수정 또는 재진단 정책 정의

### 2. 설문 데이터 모델

- `survey_results` 문서 구조 확정
- 설문 원본 응답, 총점, 위험도, 영역별 점수 필드 표준화
- 제출 시각 외에 설문 버전, 클라이언트 버전 등 메타데이터 필요 여부 검토
- 사용자별 최신 설문 결과 조회 헬퍼 정리

### 3. 맞춤 루틴 추천 로직

- 정적 루틴 3개를 사용자별 추천 구조로 전환
- 설문 결과 기반 추천 규칙 정의
- 위험도와 영역별 점수에 따른 추천 우선순위 규칙 정의
- 추천 근거 텍스트 생성 방식 정의
- 기본 루틴 fallback 정책 정의

### 4. 루틴 데이터 모델

- 루틴 정의와 사용자 배정 루틴을 분리할지 결정
- 루틴 필드 정의
- 필수 후보:
  - `routine_id`
  - `title`
  - `category`
  - `description`
  - `recommendation_reason`
  - `frequency`
  - `priority`
  - `active`
- 완료 기록 컬렉션과 추천 루틴 컬렉션 책임 분리

### 5. 루틴 API 확장

- 오늘의 루틴 조회 외에 사용자 루틴 생성 API 또는 내부 서비스 추가
- 루틴 완료 처리 API 추가
- 루틴 완료 취소 API 추가
- 주간 루틴/이력 조회 API 추가
- 설문 완료 후 루틴 자동 재생성 또는 갱신 처리 추가

### 6. 마이페이지 집계 로직

- 오늘 루틴 완료 수를 실제 루틴 완료 데이터로 계산
- 주간 달성률 계산 기준 정의
- 설문 결과를 기반으로 설문 탭 설명과 배너를 더 정교화
- 최근 활동에 설문, 루틴, 훈련 이벤트를 함께 정렬해서 제공

### 7. 인증/정책

- 설문 저장을 로그인 필수로 유지할지 검토
- 비회원 설문 결과 임시 저장 전략 검토
- 재로그인 후 임시 설문 결과 복구 여부 검토

### 8. 테스트

- 설문 저장 API 테스트 추가
- 설문 점수 검증 테스트 추가
- 루틴 추천 규칙 테스트 추가
- 루틴 완료 처리 테스트 추가
- 마이페이지 요약 집계 테스트 추가

## 권장 구현 순서

1. 백엔드 설문 점수 검증 로직 추가
2. 사용자별 맞춤 루틴 데이터 모델 설계
3. 설문 결과 기반 루틴 추천 서비스 구현
4. 루틴 완료 API 및 프론트 완료 UI 구현
5. 마이페이지 집계 고도화
6. 브라우저 E2E 검증 및 예외 처리 보강

## 메모

- 현재 루틴은 맞춤 추천이 아니라 정적 예시 성격이 강하다.
- 현재 설문 저장은 동작하지만, 서버 검증과 추천 연결은 아직 부족하다.
- 기능 완성 기준은 "설문 완료 -> 결과 저장 -> 맞춤 루틴 생성 -> 루틴 수행 -> 마이페이지 반영" 흐름이 끊기지 않는 상태다.
